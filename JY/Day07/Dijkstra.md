## 📚 최단 경로(Shortest Path) 알고리즘이란?

- **그래프에서 두 지점 사이의 가장 짧은 경로를 찾는 알고리즘**이다.
- 일반적으로 **그래프(Graph)** 로 표현하며,
  - 지점 → **노드(Node)**
  - 지점 간 연결(도로) → **간선(Edge)**
- 대표적인 최단 경로 알고리즘은 다음과 같다.
  - **(1) 다익스트라(Dijkstra)**: 한 지점에서 다른 특정 지점들까지의 최단 경로
  - **(2) 플로이드–워셜(Floyd–Warshall)**: 모든 지점에서 모든 지점까지의 최단 경로

---

### 1. 다익스트라(Dijkstra) 최단 경로 알고리즘

- **하나의 출발 노드에서 시작하여 다른 모든 노드까지의 최단 거리**를 구하는 알고리즘이다.
- **음의 가중치(음의 간선)가 있을 경우 사용할 수 없다.**
- **그리디(Greedy) 알고리즘**에 해당한다.
  - 매 단계마다 **현재 가장 비용이 적은 노드**를 선택해 나가기 때문이다.

#### 📌 다익스트라 알고리즘 수행 과정

1. **출발 노드를 설정**한다.
2. **최단 거리 테이블을 초기화**한다.
   - 모든 값을 무한(INF, 예: `int(1e9)`)으로 설정
3. **방문하지 않은 노드 중에서 최단 거리가 가장 짧은 노드**를 선택한다.
4. 해당 노드를 거쳐 다른 노드로 가는 비용을 계산하고,
   **더 짧은 경로라면 최단 거리 테이블을 갱신**한다.
5. **(3)과 (4)를 반복**한다.

- 한 번 선택된 노드는 **최단 거리가 확정**되며,
  이후에는 그 값이 더 줄어들지 않는다.
- 즉, **단계마다 하나의 노드에 대한 최단 거리가 결정**된다.

---

### 2. 간단한 다익스트라 알고리즘

- **시간 복잡도:** O(V²)  
  (V: 노드의 개수)
- 각 노드까지의 최단 거리를 저장하는 **1차원 리스트**를 사용한다.
- 매 단계마다  
  **방문하지 않은 노드 중 최단 거리가 가장 짧은 노드를 찾기 위해**
  → **모든 노드를 순차 탐색**한다.
- 노드 수가 많아질수록 성능이 급격히 저하된다.

---

### 3. 개선된 다익스트라 알고리즘

- **최단 거리가 가장 짧은 노드를 찾는 과정**을  
  **우선순위 큐(Priority Queue)** 로 대체한 방식이다.
- **시간 복잡도:** O(E log V)  
  (E: 간선의 개수, V: 노드의 개수)
- 힙(Heap) 자료구조를 사용하여,
  출발 노드로부터 **가장 가까운 노드를 빠르게 선택**할 수 있다.
- 파이썬에서는 `heapq` 라이브러리를 사용한다.
  - **최소 힙(Min Heap)**: 값이 작은 노드부터 먼저 처리
- 우선순위 큐는 **“현재까지 발견된 가장 짧은 거리의 노드”를 관리하기 위한 목적**으로 사용된다.
